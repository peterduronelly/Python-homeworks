What Makes A Video Game Successful?

*** - A dig into the world of video nerds - ***

Peter Duronelly, MSc of BA, Fall, 2017

This project is a part of a larger project by Anna Hancsák of CEU Center for Network Science which is trying to find those network properties which make teams and their products successful.

The subject of this particular research is the world of video games. This product was picked because of the handy dataset of www.mobygames.com where game enthusiast (mostly nerds) collect and contribute information on video games in a structured format.

My part of the project is far from finished. The following short presentation describes what I used from what I had learned in Scientific Python. As you will see, I did not need to use every class, but I have (sometimes extensively) used the following tools and features:

  • lists, dictionaries (a lot!), pandas
  • web scraping
  • error handling / 'trycatch'
  • network buidling
  • matplotlib

Note: I am not a game enthusiast in any sense, but the project itself is interesting.

The datasource¶

All data is coming from www.mobygames.com.

Every game has a page where game description, list of major features, developer info, and review & rating contributions are found.

Some the data had already been collected before I started my project in the following structure:

  1. file: developer_id, game_id, platform_id, title_original
  2. file: developer_id, developer_name
  3. file: game_id, game_name
  4. file: game_id platform_id, release_date, moby_rank, moby_score, publisher, developer, genre, perspective, visual, gameplay, interface, narrative
  5. file: a list of the games's url snippets (eg: "/game/windows/007-blood-stone/credits")

What we missed:

  • scores given by 'professional' critics (aka nerds): average score and the number of reviews by games and platforms
  • 'novelty score': how much a game is different from those games which have been released within a certain period before its release date

The earliest game in the dataset comes from 1966 (!), the latest is from 2017.

Each game has reviews with text and a numeric score. I needed to scrape the pages for these score. As the pages are well structured, the scores (point grades) are straightforward to get. The only problem is that sometimes:

  • a game is not scored and reviewed at all,

Determining the level of novelty for a game¶

Each game has something from the list of features. These features can be mutually exclusive (like perspective) or optional (eg gameplay).

I made the union of these features to be able to define a feature vector for all games.

  • Input: a csv file with the game and platform ids, release dates, and the features included in the game.
  • Output: a list of all features (strings) found in the dataset. The vector has exactly 100 elements.

Every game is a point in this 100-dimensional space, where all dimensions can take a value of either zero or one.

Next I defined the average vector values for each year: that is the 'average' feature of the games released in that particular year.

  • Input: a csv file with the game and platform ids, release dates, and the features included in the game.
  • Output: a dictionary, where keys are years, values are 100-element vectors with values of 0 and 1.

annual_avg_feature_vector = {}
for i in range(1966,2018):
    annual_avg_feature_vector[i] = [x * 0 for x in range(0,100)]
games_per_year = {}
for i in range(1966,2018):
    games_per_year[i] = 0

with open('title.csv') as csvfile:
    readCSV = csv.reader(csvfile, delimiter=',')
    rowcount = 0
    for row in readCSV:
        if rowcount > 0:
            for y in range(1966, 2018):
                year = row[2].split("-")[0]
                year = int(year)
                game_features = []
                if year == y:
                    for i in range(7,13):
                        text = row[i].split(",")
                        for m in range(0, len(text)):
                            f = text[m]
                            if f != "null" and f not in game_features:
                                game_features.append(f)
                    j = 0
                    for k in features_list:
                        if k in game_features:
                            annual_avg_feature_vector[y][j] = annual_avg_feature_vector[y][j] +1
                        j = j + 1
                    games_per_year[y] = games_per_year[y] + 1
        rowcount = rowcount + 1
for y in range(1966, 2018):
    if games_per_year[y] > 0:
        annual_avg_feature_vector[y] = [element / games_per_year[y] for element in annual_avg_feature_vector[y]]

And now comes the novelty score for a game: a game's degree of novelty is the cosine of the angle it closes with the average game in this multidimensional space. In theory, cosine similarity "ranges from −1 meaning exactly opposite, to 1 meaning exactly the same, with 0 indicating orthogonality (decorrelation), and in-between values indicating intermediate similarity or dissimilarity" (wikipedia). In our case, values are expected to be between (0,1).

**Cosine similarity**

  • Input: a csv file with the game and platform ids, release dates, and the features included in the game.
  • Output: a dataframe of panda dataframe of game id, platform id, cosine similarity

cosines = pd.DataFrame(columns = ["game_id", "platform_id", "cosine_similarity"])


with open('title.csv') as csvfile:
    readCSV = csv.reader(csvfile, delimiter=',')
    count = 0
    rows_in_dataframe = 0 # the actual number of rows we are having in the dataframe; we start with and empty frame
    for row in readCSV:
        if count > 0:
            clist = [] # temp list for game_id, platform_id, similarity
            game_id = row[0]
            platform_id = row[1]
            year = row[2].split("-")[0]
            year = int(year)
            game_features = [] # temp list for the game's features; as long as the number of features
            vector = [] # feature vector of 1s and 0s of the game; length: always 100
            for i in range(7,13):
                text = row[i].split(", ")
                for j in range(0, len(text)):
                    f = text[j]
                    if f != "null" and f not in game_features:
                        game_features.append(f)
            for k in features_list:
                if k in game_features:
                    vector.append(1)
                else:
                    vector.append(0)
            clist.append(game_id)
            clist.append(platform_id)
            try:
                sim = 1 - spatial.distance.cosine(annual_avg_feature_vector[year], vector) # there are 17 zero vectors 
            except RuntimeWarning:
                clist.append(1)
            else:
                clist.append(sim)
            cosines.loc[rows_in_dataframe] = clist
            #print(clist, " ", rows_in_dataframe)
            rows_in_dataframe = rows_in_dataframe + 1
        count = count + 1

nodes = []

for element in JB_game_ids:
    for i in platform_ids:
        nodes = []
        for row in df_JB_developers.itertuples(index=True, name='Pandas'):
            if getattr(row, "game_id") == element and getattr(row, "platform_id") == i :
                nodes.append(int(getattr(row, "developer_id")))
        if len(nodes) > 1:
            edges = combinations(nodes, 2)
            nx_JB_developers.add_nodes_from(nodes)
            nx_JB_developers.add_edges_from(edges)
